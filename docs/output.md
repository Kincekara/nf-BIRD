# Kincekara/nf-BIRD: Output

## Introduction

## Pipeline overview

The nf-BIRD pipeline is built using [Nextflow](https://www.nextflow.io/) and performs bacterial identification and antimicrobial resistance detection from paired-end FASTQ sequencing data. The workflow includes quality control, genome assembly, taxonomic classification, and resistance gene prediction. Key analyses encompass genome quality assessment, MLST typing, plasmid detection, AMR detection via AMRFinderPlus, and comprehensive reporting of results for each sample.

### Pipeline information

The nf-BIRD pipeline produces comprehensive output organized in the following main categories:

- **results.tsv**: A tab-delimited summary table that aggregates all important statistics and findings for every sample, including taxonomic classifications, QC metrics, AMR predictions, and MLST results.
- **Sample Reports**: Individual detailed reports for each sample are available in multiple formats:
  - Basic report: Essential findings and key metrics
  - Extended report: Comprehensive analysis results with detailed breakdowns
  - Plain text report: Machine-readable format for programmatic access
  - QC summary: Quality control metrics and assessment results
- **Tool-specific outputs**: Intermediate files from each analysis step (quality control, assembly, MLST, plasmid detection, AMR detection) are preserved in separate program directories for detailed inspection and troubleshooting.
- **Pipeline metadata**: Execution reports, DAG visualization, software versions, parameters, and samplesheet validation files are stored in the `pipeline_info/` directory.

<details markdown="1">
<summary>Output files</summary>

- `pipeline_info/`
  - Reports generated by Nextflow: `execution_report.html`, `execution_timeline.html`, `execution_trace.txt` and `pipeline_dag.dot`/`pipeline_dag.svg`.
  - Reports generated by the pipeline: `pipeline_report.html`, `pipeline_report.txt` and `software_versions.yml`. The `pipeline_report*` files will only be present if the `--email` / `--email_on_fail` parameter's are used when running the pipeline.
  - Reformatted samplesheet files used as input to the pipeline: `samplesheet.valid.csv`.
  - Parameters used by the pipeline run: `params.json`.

</details>

[Nextflow](https://www.nextflow.io/docs/latest/tracing.html) provides excellent functionality for generating various reports relevant to the running and execution of the pipeline. This will allow you to troubleshoot errors with the running of the pipeline, and also provide you with other information such as launch commands, run times and resource usage.
